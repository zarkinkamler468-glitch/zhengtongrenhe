# 教育政策智能监测平台 — 部署与使用文档

政务教育官网监测、采集、AI 解读与多租户管理。

**默认账号**：`admin` / `admin123`（上线后务必修改）

---

## 一、项目结构

```
zhengtongrenhe/
├── backend/                 # FastAPI 后端
│   ├── app/                 # 业务代码
│   ├── requirements.txt     # Python 依赖
│   ├── .env.example         # 环境变量模板
│   └── data/                # 运行时附件（自动创建）
├── frontend/                # Next.js 前端
│   ├── src/
│   ├── public/
│   └── .env.example
├── deploy/
│   └── supervisor.conf      # 进程守护配置模板
└── docs/
    └── 部署文档.md          # 本文档
```

## 三、宝塔安装软件

在 **软件商店** 安装并启动：

1. **Nginx**
2. **MySQL**（5.7 或 8.0）
3. **Redis**
4. **Supervisor 管理器**（或「进程守护管理器」）
5. **PM2 管理器**
6. **Node.js 版本管理器** → 安装 **Node 20**

**系统** → **Swap** → 添加 **2048MB**（2G 内存机器建议必做）。

## 六、配置后端

```bash
cd /www/wwwroot/zhengtongrenhe/backend
cp .env.example .env
```

编辑 `backend/.env`：

```env
DATABASE_URL=mysql+aiomysql://edu_policy:你的MySQL密码@127.0.0.1:3306/edu_policy_db?charset=utf8mb4
REDIS_URL=redis://127.0.0.1:6379/0
CELERY_EAGER=false
SECRET_KEY=用下面命令生成的随机串
ACCESS_TOKEN_EXPIRE_MINUTES=1440

LLM_PROVIDER=deepseek
LLM_API_KEY=sk-你的DeepSeek密钥
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat

ATTACHMENT_DIR=./data/attachments
DEFAULT_CRAWL_INTERVAL_MINUTES=30
```

生成 SECRET_KEY：

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(48))"
```

创建数据目录：

```bash
mkdir -p /www/wwwroot/zhengtongrenhe/backend/data/attachments
chown -R www:www /www/wwwroot/zhengtongrenhe/backend/data
```

## 九、Supervisor 守护后端（3 个进程）

打开 `deploy/supervisor.conf`，将路径 `/www/wwwroot/zhengtongrenhe` 改成你的实际路径。

在宝塔 **Supervisor 管理器** 中添加 3 个守护进程（运行目录均为 `backend/`，用户 `www`）：

| 名称 | 启动命令 |
|------|----------|
| edu-policy-api | `.venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 1` |
| edu-policy-celery | `.venv/bin/celery -A app.tasks.celery_app worker --loglevel=info --concurrency=1` |
| edu-policy-beat | `.venv/bin/celery -A app.tasks.celery_app beat --loglevel=info` |

也可直接参考项目 `deploy/supervisor.conf` 复制配置。

全部设为 **自动启动、自动重启**。

验证：

```bash
curl http://127.0.0.1:8000/api/v1/auth/config
```

应返回 JSON。

## 十一、Nginx 网站与反向代理

**网站** → **添加站点** → 填写域名 → 不创建 PHP。

站点 **配置文件** 中加入：

```nginx
client_max_body_size 50m;

location /api/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 300s;
}

location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

**SSL** → Let's Encrypt → 强制 HTTPS。

若启用 HTTPS 后前端 API 不通，修改 `frontend/.env.production` 为 `https://域名` 并重新 `npm run build`，再 `pm2 restart edu-policy-web`。

## 十五、常见问题

| 现象 | 处理 |
|------|------|
| 502 | 检查 Supervisor / PM2 进程是否在运行 |
| 能开页不能登录 | `NEXT_PUBLIC_API_URL` 与浏览器地址不一致，改后重新 build 前端 |
| 采集无反应 | 确认 Redis 已启动，Celery worker 在运行，`CELERY_EAGER=false` |
| 数据库报错 | 检查 `DATABASE_URL` 用户名密码、MySQL 是否本地可连 |
| 内存不足 | 加 Swap；MySQL `innodb_buffer_pool_size` 调到 128M |
