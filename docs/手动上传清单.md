# 手动上传清单（纯代码，约 20～50 MB）

上传到服务器目录建议：`/www/wwwroot/zhengtongrenhe/`

---

## 必须上传

### 项目根目录

```
.gitignore
README.md
docker-compose.prod.yml      ← 宝塔 Docker 部署用这个
docker-compose.yml           ← 可选
setup.bat / start.bat 等     ← 可选，服务器 Linux 用不到
scripts/dev.ps1              ← 可选
docs/                        ← 文档，可选但建议带上
```

### backend/（整包上传，但排除下面「不要」项）

```
backend/app/                 ← 全部
backend/scripts/             ← 全部（output/ 里产物可不带）
backend/Dockerfile.prod      ← Docker 部署必须
backend/Dockerfile           ← 可选
backend/requirements.txt
backend/requirements-local.txt
backend/.env.example         ← 模板，上传后在服务器复制成 .env
```

### frontend/（整包上传，但排除下面「不要」项）

```
frontend/src/                ← 全部
frontend/public/             ← 全部（含地图图片）
frontend/Dockerfile.prod     ← Docker 部署必须
frontend/Dockerfile          ← 可选
frontend/package.json
frontend/package-lock.json
frontend/next.config.ts
frontend/tsconfig.json
frontend/postcss.config.mjs
frontend/tailwind.config.ts
frontend/next-env.d.ts
frontend/.env.example
```

---

## 不要上传（占空间 / 含密钥 / 服务器会重建）

| 路径 | 原因 |
|------|------|
| `frontend/node_modules/` | ~430 MB，服务器 build 时 npm install |
| `frontend/.next/` | ~490 MB，构建缓存 |
| `frontend/.env.local` | 本地配置，可能含地址 |
| `backend/.venv/` | ~116 MB，服务器/Docker 里重装 |
| `backend/data/` | 本地附件与数据库文件 |
| `backend/.env` | **含密码和 API Key，禁止上传** |
| `backend/scripts/output/*` | 调试脚本产物 |
| `**/__pycache__/` | Python 缓存 |
| `**/*.pyc` | Python 缓存 |
| `.idea/` `.vscode/` | IDE 配置 |

---

## 服务器上上传后要做

1. `cp backend/.env.example backend/.env` 并填写数据库、Redis、LLM 密钥  
2. 根目录创建 `.env`：`NEXT_PUBLIC_API_URL=https://你的域名`  
3. `docker compose -f docker-compose.prod.yml up -d --build`  
4. 宝塔 Nginx 反代 3000 / 8000  

详见 [宝塔部署-精简Docker.md](./宝塔部署-精简Docker.md)

---

## 一键打包包（Windows）

在项目根目录 PowerShell 执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\pack-upload.ps1
```

会在桌面生成 **`zhengtongrenhe-upload.zip`**，只含应上传文件，直接上传到宝塔即可。
