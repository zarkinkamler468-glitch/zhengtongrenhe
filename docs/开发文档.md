## 一、项目背景

随着教育政策、科研项目、专项资金、职业教育改革、产教融合等信息发布频率不断增加，高校、职业院校、教育集团及教育科技企业需要及时掌握最新政策动态。

目前信息分散在教育部、省教育厅、高校官网、科研项目平台等多个渠道，人工查找效率低、容易遗漏重要通知。

因此建设“教育政策智能监测平台”，实现：

* 自动监测教育类官方网站
* 自动采集政策与通知
* AI自动摘要与解读
* 智能分类与标签管理
* 关键词订阅与消息推送
* 教育政策知识库建设

---

# 二、建设目标

建设一套面向教育行业的信息监测与分析平台，实现：

1. 自动发现教育政策更新
2. 自动采集公开信息
3. AI自动解读政策内容
4. 构建教育政策知识库
5. 实现政策预警推送
6. 提供统一检索入口

# 三、建设原则

## 合规原则

* 教育部官网
* 省教育厅官网
* 高校官网

* 政府公开信息平台

禁止采集：

* 登录后内容
* 内部管理系统
* 非公开数据

## 快速启动（本地，无 Docker）

**最简单：双击项目根目录 `setup.bat`（首次）或 `start.bat`**

| 文件 | 作用 |
|------|------|
| `setup.bat` | 首次安装依赖 |
| `start.bat` | 一键启动前后端 |
| `stop.bat` | 一键停止 |
| `restart.bat` | 重启 |

或在 PowerShell 中（需加 Bypass，否则可能报「禁止运行脚本」）：

```powershell
cd d:\桌面\zhengtongrenhe
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev.ps1 setup
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev.ps1 start
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev.ps1 stop
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\dev.ps1 restart
```

- 前端：http://localhost:3001
- 后端文档：http://localhost:8000/docs

## DeepSeek 配置

```env
LLM_PROVIDER=deepseek
LLM_API_KEY=sk-xxx
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
EMBEDDING_MODEL=          # 留空，DeepSeek 无 embedding 接口
```

在 [DeepSeek 开放平台](https://platform.deepseek.com/) 申请 API Key 后填入 `backend/.env`。

## 核心业务闭环

```
监测源/栏目配置 → Celery 定时巡检 → 内容采集入库
    → AI 摘要/标签/解读 → Elasticsearch/PostgreSQL 索引
    → 关键词匹配推送 → API 检索/问答
```

## 主要 API 端点

| 模块 | 端点 | 说明 |
|------|------|------|
| 认证 | `POST /api/v1/auth/login` | 登录获取 Token |
| 监测源 | `GET/POST /api/v1/monitor/sources` | 网站管理 |
| 栏目 | `GET/POST /api/v1/monitor/columns` | 栏目管理 |
| 采集 | `POST /api/v1/monitor/columns/{id}/crawl` | 手动触发采集 |
| 政策库 | `GET /api/v1/articles` | 政策列表 |
| 检索 | `GET /api/v1/articles/search?q=职业教育` | 全文搜索 |
| 详情 | `GET /api/v1/articles/{id}` | 含 AI 解读 |
| 订阅 | `POST /api/v1/subscriptions` | 关键词订阅 |
| 统计 | `GET /api/v1/analytics/overview` | 数据分析 |
| 问答 | `POST /api/v1/qa/ask` | AI 政策问答 |
| 日志 | `GET /api/v1/logs/crawl` | 采集日志 |

## 采集引擎说明

- 支持静态页面（httpx）与动态页面（Playwright）
- 栏目可配置 CSS 选择器：`list_selector`、`title_selector`、`content_selector`、`date_selector`
- 自动识别 PDF/DOC/DOCX/XLS/XLSX 附件并下载
- 巡检频率：5/15/30/60 分钟（栏目级配置）

```bash

uvicorn app.main:app --reload --port 8000
celery -A app.tasks.celery_app worker -l info
celery -A app.tasks.celery_app beat -l info

# 前端
cd frontend && npm install
cp .env.example .env.local
npm run dev   # http://localhost:3000
```
